# Chu-kei Quality Gates

Chu-keiでは、企業数の増加よりもデータ状態の透明性と再検証可能性を優先する。

## 1. データ深度ゲート

### 本番

- 公式一次資料を確認済み
- 中計または中計相当資料の区分が明確
- 経営課題、成長戦略、財務目標、資本政策、株主還元を確認
- 数値目標の年度・単位を確認
- 証跡を保持
- 未確認事項を明示

### 詳細抽出済みβ

- 公式一次資料を確認済み
- 主要論点を構造化済み
- 詳細ページ番号、最新実績、計画改定履歴の一部が未完了でもよい
- 不足項目を明示

### 一次確認β

- 上場状態、証券コード、市場、業種を確認
- 会社公式IRの確認入口を特定
- 中計本文の詳細抽出前であることを明示

### カバレッジβ

- JPX上場情報、市場、業種を確認
- 中計または中計相当資料の特定は未完了
- 中計開示企業と誤認させない

## 2. 出典ゲート

- 会社公式IR、公式PDF、JPX資料を優先する
- ミラーサイトや二次情報は一次資料への到達補助に限定する
- URL、資料名、公表日または確認日、ページ番号を可能な限り保持する
- リンク切れを即座に「資料なし」と判定しない

## 3. 日付ゲート

`publicationDate`と`verifiedAt`を分離する。

- `publicationDate`: 企業が資料を公表した日
- `verifiedAt`: Chu-keiが資料・URL・上場情報を確認した日
- 確認日を中計公表日のように表示しない
- 公表日不明の場合は未確認として扱う

## 4. 表示・比較ゲート

- 企業探索カバレッジと中計分析カバレッジを分離表示する
- 異なるデータ深度の企業を比較する場合、各社の状態を常に表示する
- 未抽出項目を「なし」と表示しない
- 推測値、算定値、会社開示値を区別する
- スマホで検索・一覧・比較・詳細の主要導線を利用できる

## 5. リリースゲート

- 証券コード重複なし
- 全JSON構文正常
- JavaScript構文正常
- DOM参照先が存在
- 相対アセット欠損なし
- ZIP再展開後に同じ検証が通る
- SHA-256を記録
- ブラウザ実動作QAを記録

## 6. 禁止事項

- 公式資料にない数値を確定値として登録しない
- 未確認情報を空欄埋めのために推測しない
- カバレッジβを中計読込済み企業として数えない
- 投資判断を煽る表現を使用しない
- 掲載順位や評価を販売しない

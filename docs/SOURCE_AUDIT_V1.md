# Chu-kei 出典・リンク監査 v1

## 目的

公式資料への入口を継続的に確認し、リンク切れとアクセス制限を区別しながら、修正すべき企業を特定する。

## 二段階の監査

### 1. 変更時の決定論的監査

PRとmain更新時に必ず実行する。

- ソース確認済み企業が200社であること
- URLが存在し、構文解析できること
- HTTPSであること
- 認証情報やローカル・プライベートホストを含まないこと
- 最終確認日がISO日付であること
- 資料名が登録されていること
- 重複URL、URLフラグメント、過度に複雑なクエリを警告として記録すること

重複URLはIRトップページを共有する場合があるため、自動エラーにはしない。

### 2. 週次の実URL監査

毎週、重複を除いたURLを確認する。

- `ok`: 2xx / 3xx
- `restricted`: 401 / 403。リンク切れとは扱わない
- `missing`: 404 / 410。修正候補
- `rate_limited`: 429。再確認対象
- `server_error`: 5xx。継続監視対象
- `network_error`: タイムアウト、DNS、TLS等。再確認対象

HEADが拒否された場合はGETへ切り替え、ネットワークエラー・429・5xxは一度だけ再試行する。

## 公開判断

外部サイトの一時障害でPRを不安定にしないため、実URL監査はPRの必須ゲートにしない。構造不正は公開を止め、死活結果はArtifactとして保存し、`missing`を改善キューへ送る。

## 秘密情報

URL、レスポンス状態、最終遷移先、応答時間のみを記録する。Cookie、認証情報、本文は保存しない。

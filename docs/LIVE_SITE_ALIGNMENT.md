# Chu-kei 公開サイトとGitHubの同期方針

## 対象

- 公開サイト: `https://chukei-insight.osugimurata.chatgpt.site/`
- 表示名: `Chu-kei`
- GitHub: `M-Osugi1230/chu-kei`

## 正本の役割

### 公開サイト

公開サイトは、ユーザー体験、画面構成、説明文、探索導線、比較体験、将来の有料導線についての最新プロダクト基準とする。

### GitHub

GitHubは、企業データ、出典、品質状態、監査結果、変更履歴、レビュー判断、昇格判断、再現可能な検証コードについての正本とする。

公開サイト上の表示変更がデータの意味を変える場合、GitHubのSchema、監査、変更台帳を先に更新する。公開サイトだけで企業データを修正しない。

## 現在の公開サイト基準

2026-07-11時点で、別のプロダクト開発スレッドから次の状態が共有されている。

- 公開版: v15
- 詳細抽出済みβ: 70社
- 公表日監査を充足: 59社
- 精査中: 11社
- 企業別変更履歴: 14件

この状態はサイト側の進捗報告であり、GitHubへ同じ監査台帳と変更履歴が取り込まれるまでは、GitHubの公開データを自動更新しない。

## 同期単位

公開サイトの各リリースについて、`operations/site-sync/`に次を記録する。

- リリース番号
- 公開URL
- 確認日
- 主要機能
- データ品質指標
- GitHubへ取り込み済みの項目
- 未同期項目
- 根拠となる監査・変更履歴ファイル

## 同期ゲート

公開サイトの変更をGitHubの正本へ反映するには、次を満たす。

1. 公式一次資料の出典がある。
2. 資料公表日と確認日が分離されている。
3. 未確認・未開示・未抽出を推測で補完していない。
4. 修正履歴またはレビュー判断が記録されている。
5. Data Contract Gateが成功する。
6. strict品質監査が成功する。
7. 検索・比較・モバイル表示の回帰テストが成功する。

## 収益化との関係

公開サイトは無料の比較・理解体験を磨き、GitHubの品質済みデータを将来のスポット調査レポート、Pro、Team、API・データライセンスへ接続する。

収益化のために品質状態や出典制約を非表示にしない。Chu-keiは投資助言または銘柄推奨を目的としない。

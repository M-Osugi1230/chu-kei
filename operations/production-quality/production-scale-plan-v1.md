# 本番品質1,000社 拡大計画 v1

## 原則

- 公式一次資料のみを使用する。
- ページ番号付き証跡を必須とする。
- 未開示値を推計しない。
- 自動選択・自動承認・自動本番昇格を禁止する。
- 個別承認と独立再検証を企業単位で記録する。
- 品質改善時は品質負債の許容上限を同時に引き下げる。

## 現在地

- 掲載企業: 1,200社
- 本番ステージ: 126社
- 最高品質5つ星: 112社
- 詳細抽出済みβ: 96社
- 一次確認β: 76社
- Coverageβ: 902社

## 拡大順序

1. 既存coreの品質負債14社を解消する。
2. 詳細抽出済みβ96社を不足類型ごとに補修する。
3. 一次確認β76社を詳細構造化・進捗接続・二者承認する。
4. Coverageβから公式中計を持つ企業を発見し、同じ契約で昇格する。

## 大量処理の安全策

- 候補抽出と承認を分離する。
- 候補はコード明示方式で固定する。
- 正本変更前後の社数、品質分布、容量を検証する。
- ブラウザE2E・アクセシビリティ・静的容量予算を必須とする。
- 初期ロード上限に近づく前にインデックス項目を再設計する。

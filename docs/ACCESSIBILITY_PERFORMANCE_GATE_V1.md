# アクセシビリティ・性能ゲート v1

## 目的

検索・比較・品質確認を、デスクトップとスマホの双方で迷わず利用できる状態をCIで維持する。

## アクセシビリティ

Playwrightとaxe-coreで、トップ画面と品質ダッシュボードを次の条件で確認する。

- WCAG 2.0 / 2.1 A・AA
- スキップリンクとキーボードフォーカス
- フォームラベル、見出し、テーブル、ダイアログ
- デスクトップ1440px
- モバイル390px
- コンソールエラー・ページエラー0件
- スマホの操作対象が24px未満にならないこと

## 性能

外部サービスの変動を受けない決定論的予算と、実ブラウザ上の操作予算を分離する。

### 静的予算

- HTML・CSS・JavaScriptのサイトシェル: 160KB以下
- HTML合計: 40KB以下
- JavaScript合計: 90KB以下
- CSS合計: 45KB以下
- 圧縮企業データ: 100KB以下
- データチャンク: 64個以下
- 外部JavaScript・外部CSSへの依存なし
- `eval`と`document.write`を禁止

### 操作予算

- 570社の初期表示: 8秒以内
- 検索結果反映: 1.5秒以内
- 戦略フィルター反映: 1.5秒以内
- 品質レビューキュー表示: 8秒以内

CI環境の変動を考慮して余裕を持たせ、継続的な悪化を検知する。性能予算を緩和する場合は、理由と計測結果をPRへ記録する。

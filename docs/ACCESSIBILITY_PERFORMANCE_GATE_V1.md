# アクセシビリティ・性能ゲート v1

## 目的

検索・比較・品質確認を、デスクトップとスマホの双方で迷わず利用できる状態をCIで維持する。

## アクセシビリティ

Playwrightとaxe-coreで、トップ画面と品質ダッシュボードを次の条件で確認する。

- WCAG 2.0 / 2.1 A・AA
- スキップリンクとキーボードフォーカス
- フォームラベル、見出し、テーブル、ダイアログ
- デスクトップ1440px
- モバイル390px
- コンソールエラー・ページエラー0件
- スマホの操作対象が24px未満にならないこと
- 本番品質件数は固定値を使わず、正本JSONの現在値と画面表示の一致を確認する

## 性能

外部サービスの変動を受けない決定論的予算と、実ブラウザ上の操作予算を分離する。

### 静的予算

- HTML・CSS・JavaScriptのサイトシェル: 180KB以下
- HTML合計: 40KB以下
- JavaScript合計: 110KB以下
- CSS合計: 45KB以下
- 正規化済み圧縮データ:
  - 全体の絶対上限512KB
  - 構造化済み1社あたり1,500バイト以下
  - データチャンク256個以下
- 初期表示用データ:
  - 掲載2,500社・本番品質2,000社を一覧検索、戦略フィルター、カード要約、進捗件数付きで表示するため192KB以下
  - 長文、数値詳細、注意事項、原文証跡は企業詳細シャードへ分離する
- 企業詳細データ:
  - 20社単位で遅延読込
  - 各圧縮シャード32KB以下
- 外部JavaScript・外部CSSへの依存なし
- `eval`と`document.write`を禁止

初期表示用データの上限は、単に掲載社数に比例して緩和しない。検索・フィルター・カード表示に必要な情報だけを初期データへ残し、長文、数値詳細、注意事項、原文証跡は企業詳細シャードへ分離する。正規化済み全体データの絶対上限、1社当たり密度、詳細シャード上限も同時に検証する。

### 操作予算

- 掲載2,500社の初期表示: 8秒以内
- 検索結果反映: 1.5秒以内
- 戦略フィルター反映: 1.5秒以内
- 品質レビューキュー表示: 8秒以内

CI環境の変動を考慮して余裕を持たせ、継続的な悪化を検知する。性能予算を変更する場合は、理由、計測結果、効率性を担保する代替条件をPRへ記録する。

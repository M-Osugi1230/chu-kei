# アクセシビリティ・性能ゲート v1

## 目的

検索・比較・品質確認を、デスクトップとスマホの双方で迷わず利用できる状態をCIで維持する。

## アクセシビリティ

Playwrightとaxe-coreで、トップ画面と品質ダッシュボードを次の条件で確認する。

- WCAG 2.0 / 2.1 A・AA
- スキップリンクとキーボードフォーカス
- フォームラベル、見出し、テーブル、ダイアログ
- デスクトップ1440px
- モバイル390px
- コンソールエラー・ページエラー0件
- スマホの操作対象が24px未満にならないこと

## 性能

外部サービスの変動を受けない決定論的予算と、実ブラウザ上の操作予算を分離する。

### 静的予算

- HTML・CSS・JavaScriptのサイトシェル: 160KB以下
- HTML合計: 40KB以下
- JavaScript合計: 90KB以下
- CSS合計: 45KB以下
- 圧縮企業データ:
  - 135構造化社時点の96KBを基準とし、追加1社あたり800バイト以内の増分予算
  - 構造化済み1社あたり760バイト以下
  - 絶対上限128KB
- データチャンク: 64個以下
- 外部JavaScript・外部CSSへの依存なし
- `eval`と`document.write`を禁止

圧縮企業データは、社数増加に合わせて単純に上限を緩和しない。基準容量、1社当たり増分、1社当たり密度、絶対上限を同時に検証し、データ量の増加と配信効率を両立する。

### 操作予算

- 570社の初期表示: 8秒以内
- 検索結果反映: 1.5秒以内
- 戦略フィルター反映: 1.5秒以内
- 品質レビューキュー表示: 8秒以内

CI環境の変動を考慮して余裕を持たせ、継続的な悪化を検知する。性能予算を変更する場合は、理由、計測結果、効率性を担保する代替条件をPRへ記録する。

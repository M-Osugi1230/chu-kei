# 検索・フィルター契約 v1

## 目的

会社名、証券コード、戦略キーワード、品質階層を組み合わせても、結果の意味が安定し、将来のデータ更新で検索体験が壊れないようにする。

2026-07-22時点では、掲載2,500社すべてが本番品質であり、Coverageβは0社である。件数は固定値として検索ロジックへ埋め込まず、正本データから取得する。

## 文字正規化

- Unicode NFKCで全角・半角を統一する
- 大文字・小文字を区別しない
- 全角スペースと連続空白を単一空白へ統一する
- 証券コード4桁または英字を含む4文字コードの完全一致は、他社のURLや本文中の文字列へ誤ヒットさせない

## キーワード検索

空白で区切った複数語はAND条件とする。

例:

- `海外 ROIC`
- `7011 Prime`
- `M&A 人材`

検索対象:

- 証券コード
- 会社名
- 市場
- 業種
- カテゴリ
- 概要
- 対象資料名
- 戦略テーマ
- 主なポイント

## フィルター

以下はすべてAND条件で組み合わせる。

- キーワード
- 市場
- データ品質階層
- 戦略カード
- 保存企業のみ

## 並び順

1. 証券コード・会社名の完全一致
2. 証券コード・会社名の前方一致
3. 会社名の部分一致
4. その他のキーワード一致
5. 同順位では元データの順序を維持

## 品質階層

- 本番品質2,500社は、公式一次資料、構造化、レビュー、独立再検証を満たす企業として検索する
- Coverageβは0社であるが、将来の新規上場企業追加時に品質階層を再利用できる契約を維持する
- 品質階層を変更しても、会社名・証券コード・市場・業種の基本検索結果を欠落させない
- 上場廃止企業は現行検索インデックスへ含めず、公式理由付きアーカイブで履歴を保持する

## 2,500社完了時の検証

- 正本企業数、検索インデックス企業数、企業詳細データ数が2,500社で一致する
- 本番品質フィルターの結果が2,500社となる
- Coverageβフィルターの結果が0社となる
- 置換後企業の証券コード `3905`、`6069`、`6564` が完全一致検索できる
- 上場廃止済みの `3681`、`5903`、`7317` は現行企業検索へ表示しない
- 全角コード、複数語AND、市場、業種、戦略、保存企業、複合条件の回帰テストを維持する

## 品質原則

検索結果件数と個別企業の一致条件をCIで検証する。検索ロジックの変更時は、完全一致、全角コード、英字を含む証券コード、複数語AND、市場、品質階層、戦略、複合条件の回帰テストを通過させる。

企業数の拡張・置換時は、正本データ件数、検索インデックス件数、企業詳細件数が一致することをデータ契約と併せて確認する。

# Chu-kei 調査ワークスペース v1

この文書はIssue #23の実装方針を定義する。

## 目的

企業を一度検索するだけでなく、候補を保存し、検索条件を切り替え、複数社を比較し、同じ状態を後から再現できる調査体験を提供する。

## 利用者価値

- 気になる企業を端末内へ保存できる
- 保存企業だけに絞って再確認できる
- 品質、最終確認日、証券コードで並べ替えられる
- 検索条件・戦略・比較対象をURLで共有できる
- カード上で構造化項目の充足度と進捗接続有無を把握できる

## プライバシー

保存企業はブラウザのlocalStorageだけに保存する。外部サーバーへ送信しない。外部解析SDKも追加しない。

## URL契約

- `q`: 検索語
- `market`: 市場
- `stage`: データ品質
- `strategy`: 戦略軸
- `sort`: 並べ替え
- `saved`: 保存企業のみ表示する場合は`1`
- `compare`: 比較対象の証券コードをカンマ区切りで最大4社
- `#company=XXXX`: 企業詳細を直接開く

不正な市場・品質・戦略・並べ替え・証券コードは無視する。

## 品質条件

- 既存の570社、200社、100社、149件表示を維持
- 既存検索・戦略選択・企業詳細・比較を維持
- URLから状態を復元できる
- 保存状態が再読み込み後も維持される
- 390px幅で横スクロールしない
- WCAG A/AA違反を増やさない
- コンソールエラーとページエラーを発生させない

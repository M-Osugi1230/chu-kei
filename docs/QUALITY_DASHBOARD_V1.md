# 品質ダッシュボード v1

## 目的

掲載企業数だけではなく、確認済み範囲・不足項目・レビュー優先順位をユーザーと運営の双方に説明できる状態を作る。

## 構造化済み企業プール

本番品質と詳細抽出済みβを合わせた構造化済み企業プールを、公式ソース、公表日、ページ証跡、構造化、数値・方針抽出、進捗接続、原文証跡の7項目で機械評価する。

本番品質への昇格では、詳細抽出済みβの社数は減り、本番品質の社数が増える。そのため退行判定は詳細抽出済みβ単独ではなく、両ステージを合計した企業数で行う。最低維持数は `operations/quality-debt-budget-v1.json` の `minimumCounts.structured.reviewableCompanies` で管理する。

文書準備度は作業順を決める指標であり、本番昇格の自動判定ではない。

## 本番昇格の必須レビュー

1. 原文との数値・単位・年度突合
2. 戦略分類の妥当性確認
3. 別確認者によるダブルチェック
4. 会社詳細、比較、スマホ表示の監査

## 現在の監査基準

- 構造化済み企業プール最低維持数: 222社
- 内訳は本番品質と詳細抽出済みβの合計で判定
- 詳細抽出済みβ内の優先度A/Bはページ証跡の有無から毎回動的に算出
- 本番品質の不足項目と詳細抽出済みβのページ証跡不足は品質負債予算で上限管理

未確認項目は推測で補完せず、改善キューとして残す。

# 公式資料・証跡候補収集 v1

## 目的

本番30社の資料公表日・ページ証跡不足について、公式URLから確認候補を収集し、人手レビューの準備時間を短縮する。

## 収集内容

- HTTP状態、最終URL、Content-Type、Last-Modified、ETag
- PDFページ数とメタデータ
- PDF・HTML本文に現れる日付候補
- 中計、売上高、営業利益、ROIC、ROE、株主還元等を含むページ候補
- HTMLページから見つかったPDFリンク候補

## 厳守事項

- 収集結果は候補であり、データを自動更新しない
- HTTP Last-ModifiedやPDF CreationDateを資料公表日と断定しない
- 公式資料本文に記載された日付を人が確認する
- ページ候補の文脈、数値、単位、年度を人が確認する
- 修正は修正台帳とレビュー台帳を通す
- 自動処理が`corrected`や本番昇格を記録しない

## セキュリティ

- HTTPSのみ
- 認証情報付きURLを拒否
- localhost、プライベートIP、リンクローカルを拒否
- リダイレクト後のURLも再検査
- 1資料40MBまで
- タイムアウト20秒
- PDF本文・元ファイルはArtifactへ保存せず、候補JSONのみ保存

## 実行モード

### PR / main

`--dry-run`で対象企業とポリシーのみ検証する。外部サイトの一時障害を必須ゲートへ持ち込まない。

### 手動 / 週次

Popplerを導入し、公式資料から候補を収集する。失敗は企業単位で記録し、全体処理は継続する。

候補レポートは90日間Artifactとして保存する。

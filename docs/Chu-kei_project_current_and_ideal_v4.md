## 2. 今回の拡張

1. 第13〜17回の公式資料調査を連続実行し、未調査だった500社を網羅した。
2. 過去第1〜17回の候補レポートを統合し、現在もCoverageβである企業だけを再評価した。
3. 過去バッチ間で証券コードが重複する場合は、ステータス、企業同一性、信頼度、資料公表日、ページ数、ページ証跡数の順で、より品質の高い候補を決定論的に採用した。
4. 統合対象2,195社のうち、現在ステージ、公式PDF、企業同一性、信頼度、公表日、資料種別、複数ページ証跡、テーマ、構造化概要等の条件を満たす457社を特定した。
5. 2,000社到達に必要な451社だけを提案SHA-256で固定し、自動選定を行わず明示承認した。
6. 451社に一次レビューと独立再検証を付与し、本番品質へ昇格した。
7. 条件を満たした残り6社は、今回の承認対象数を超えるため昇格せず、次回候補として残した。

本番品質は1,549社から2,000社へ増加した。Coverageβは951社から500社へ減少し、公式資料確認率は61.96%から80.00%へ上昇した。

## 4. 品質基準

本番品質企業には次を必須とする。

- 会社公式またはJPX企業コード別の公式一次資料
- 企業名または証券コードの一致
- 資料名、公表日、最終確認日
- PDFページ番号または具体的な公式Web見出し
- 経営課題、戦略、数値、資本政策等の構造化
- 数値の単位、年度、目標・実績区分への注意
- 比較不能な場合の明示
- 一次レビュー
- 独立再検証

自動事実補完、自動選定、自動承認、自動本番昇格は禁止する。比較不能な目標に架空の進捗率を作らない。

## 5. 基盤改善

- 複数の公式資料調査バッチを番号順に連続処理できるようにした。
- 各バッチの候補生成後、その結果を次バッチの除外条件へ即時反映するようにした。
- 複数の候補レポートを統合して、一つの提案ハッシュへ固定できるようにした。
- 現在の企業ステージを提案条件に含め、すでに本番品質の企業を再昇格対象にしないようにした。
- 過去バッチの候補重複を品質順で決定論的に解決するようにした。
- Browser E2Eとアクセシビリティ監査は、本番企業数を正本JSONから動的に検証する。

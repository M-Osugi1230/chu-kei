# 公開サイトv15 データ引き継ぎ手順

## 目的

公開サイト `https://chukei-insight.osugimurata.chatgpt.site/` で進んだ調査結果を、推測や転記漏れなくGitHubの正本へ同期する。

公開サイトの画面表示やチャット内の説明だけを根拠に企業データを更新しない。必ず構造化レコードと公式一次資料を受け取る。

## 今回受け取るデータ

### 1. 詳細抽出済みβ70社の公表日監査

報告済みの状態:

- 監査充足: 59社
- 精査中: 11社
- 合計: 70社

保存先:

`operations/site-sync/incoming/chukei-insight-v15-publication-date-audit.json`

各社について必要な情報:

- 証券コード
- 会社名
- 対象資料名
- 監査状態
- 公式資料URL
- 公表日候補
- 日付粒度
- 公表日の根拠
- 抽出日
- レビュー状態
- レビュー担当とレビュー日

公表日候補はHTTPの`Last-Modified`やPDF作成日だけで確定しない。公式リリース日、資料表紙、本文、取引所開示等を証跡として記録する。

### 2. 企業別変更履歴14件

各変更について必要な情報:

- 一意な変更ID
- 証券コード
- 会社名
- 変更種別
- 変更対象フィールド
- 変更前

- レビュー担当とレビュー日

AIが変更候補を作成してもよいが、企業の方針変更として公開する前に人が意味を確認する。

## ステータス

### `awaiting_export`

件数だけ共有され、実レコードが未受領の状態。`records`は空配列とする。

### `review_ready`

必要件数の実レコードを受領し、形式・企業コード・出典を検査できる状態。公開データはまだ変更しない。

### `approved`

全レコードについて人手レビューが完了した状態。GitHubの企業データへ適用する別PRを作成できる。

## 自動更新の禁止

両インポートファイルは常に次を保持する。

```json
"automaticUpdateAllowed": false
```

同期ゲートはデータの形式と根拠を検査するが、企業データの自動変更は行わない。

## 同期フロー

1. 別チャットまたは公開サイト側からJSONを出力する。
2. `incoming/`の該当ファイルへ配置する。
3. ステータスを`review_ready`へ変更する。
4. `Chu-kei Live Site v15 Sync Contract`を実行する。
5. エラーと未確認項目を修正する。
6. 人手レビュー後に`approved`へ変更する。
7. 企業データへ適用するPRを別に作成する。
8. Data Contract、品質スコア、E2E、公開整合をすべて再実行する。

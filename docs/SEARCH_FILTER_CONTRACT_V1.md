# 検索・フィルター契約 v1

## 目的

会社名、証券コード、戦略キーワード、品質階層を組み合わせても、結果の意味が安定し、将来のデータ更新で検索体験が壊れないようにする。

## 文字正規化

- Unicode NFKCで全角・半角を統一する
- 大文字・小文字を区別しない
- 全角スペースと連続空白を単一空白へ統一する
- 証券コード4桁の完全一致は、他社のURLや本文中の数字へ誤ヒットさせない

## キーワード検索

空白で区切った複数語はAND条件とする。

例:

- `海外 ROIC`
- `7011 Prime`
- `M&A 人材`

検索対象:

- 証券コード
- 会社名
- 市場
- 業種
- カテゴリ
- 概要
- 対象資料名
- 戦略テーマ
- 主なポイント

## フィルター

以下はすべてAND条件で組み合わせる。

- キーワード
- 市場
- データ品質階層
- 戦略カード

## 並び順

1. 証券コード・会社名の完全一致
2. 証券コード・会社名の前方一致
3. 会社名の部分一致
4. その他のキーワード一致
5. 同順位では元データの順序を維持

## 品質原則

検索結果件数と個別企業の一致条件をCIで検証する。検索ロジックの変更時は、完全一致、全角コード、複数語AND、市場、品質階層、戦略、複合条件の回帰テストを通過させる。

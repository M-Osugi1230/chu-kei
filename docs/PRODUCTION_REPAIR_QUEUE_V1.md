# 本番品質データ補修キュー v1

## 目的

本番30社を一律に完成扱いせず、現在の根拠不足を動的に抽出し、原資料確認の作業順を明確にする。

## 優先度

- `P0`: 公式ソースまたは最終確認日の構造不備
- `P1`: 資料公表日が未確認
- `P2`: ページ証跡が未登録
- `P3`: 数値・方針抽出または進捗接続が不足
- `P4`: 補修項目なし

優先度はデータ補修の作業順であり、企業や計画の評価ではない。

## 出力

`scripts/build_production_repair_queue_v1.mjs` が次を生成する。

- `artifacts/production-repair-queue-v1.json`
- `artifacts/production-repair-queue-v1.csv`

キューには品質スコア、資料公表日、ページ証跡、進捗接続、ソースURL、不足項目を含める。

## 改善可能な監査

不足件数をCIの固定値にしない。原資料確認によって不足が減った場合は、キューとサマリーが自動的に更新され、改善そのものを正常として扱う。

★★★★★は8つの品質項目をすべて満たすため、補修キューには含めない。

## 修正手順

1. 補修キューから対象企業を選ぶ
2. 公式資料を開く
3. 公表日、ページ、数値、単位、年度を確認する
4. 修正台帳へ記録する
5. 別確認者がレビューする
6. データバンドルを更新する
7. 全品質ゲートとブラウザE2Eを実行する

# Chu-kei スポット調査 運用手順 v1

## 1. 受付

1. Netlify Formsで依頼を受信する。
2. 氏名、連絡先、対象企業、利用目的、希望メニューを確認する。
3. 不足情報がある場合は、見積前に対象範囲を確定する。
4. 投資助言、株価予測、未公開情報の調査依頼は受け付けない。

## 2. スコープ・見積

- 対象企業数
- 対象計画・対象期間
- 比較軸
- 必要な成果物形式
- 納品予定日
- 価格
- 修正回数

を依頼者と合意する。標準価格は `operations/commercial/offers-v1.json` を正本とする。

## 3. 調査

1. 会社公式IRページから資料を取得する。
2. 資料名、公表日、URL、ページを記録する。
3. 定性情報と定量情報を分離する。
4. 当初目標、修正目標、最新見通し、実績を区別する。
5. 不明項目は未確認・未開示・未抽出として残す。
6. `docs/SPOT_RESEARCH_REPORT_TEMPLATE_V1.md`に沿って成果物を作成する。

## 4. 品質レビュー

- 数値、年度、単位、対象範囲を原文突合
- ページ証跡確認
- 比較軸・定義差確認
- 出典リンク確認
- 投資推奨表現の除去
- 抽出者と異なる確認者によるレビュー

レビュー未完了の成果物を納品しない。

## 5. 納品

- 合意した形式で納品
- 出典一覧と品質・制約を必ず含める
- 修正受付範囲を明記
- 追加調査は別スコープとして扱う

## 6. 案件管理

案件ごとに次を記録する。

- 受付日
- 案件ID
- メニュー
- 対象企業
- ステータス
- 見積額
- 調査担当
- レビュー担当
- 納品日
- 修正回数
- 利用者フィードバック

依頼者情報と成果物は、調査・見積・納品・品質改善の目的に限定して扱う。

# 企業同一性・重複監査 v1

## 目的

証券コードの重複だけでなく、会社名の表記揺れ、旧社名、略称、英語名、法人格表記の違いを管理し、同一企業の二重登録や別企業の誤統合を防ぐ。

## 正本

- 証券コードを企業の主キーとする
- 会社名は表示名であり、同一性判定の唯一の根拠にしない
- 確認済みの旧社名・略称等は`operations/entities/aliases.json`へ登録する
- aliasには公式な出典URLと確認日を必須とする

## 自動監査

- 570社、証券コード一意
- 会社名の欠損
- 完全一致する会社名
- NFKC、法人格、空白、記号を除いた正規化名の衝突
- aliasの形式・重複・孤立参照
- aliasと別企業の正式名称の衝突

正規化名の衝突は自動削除・統合せず、警告として人が確認する。持株会社化、商号変更、同名企業等を機械的に同一視しない。

## alias種別

- former_name
- abbreviation
- english_name
- spacing_variant
- legal_suffix_variant
- other

企業名の変更を確認した場合は、旧表示名をaliasへ残し、検索互換性と変更履歴に利用する。

# Chu-kei 品質負債予算 v1

## 目的

Chu-keiでは、品質ゲートの本数ではなく、未確認・根拠不足・古い確認日・日付矛盾などの品質負債が増えていないかを継続管理する。

品質負債予算は、現時点の欠損を完成扱いする仕組みではない。現在値を上限として固定し、今後の変更で悪化することを防ぎながら、改善のたびに上限を引き下げる。

## 現在の基準値

2026-07-11時点の公開データから、次を品質負債として固定する。

- 本番30社の資料公表日不足: 4社
- 本番30社のページ証跡不足: 23社
- 本番30社の★★★★★未達: 23社
- 詳細抽出済みβ70社のページ証跡不足: 53社

上記以外の監査項目は0件を上限とする。

## 主な監査項目

### 日付整合

- `lastVerifiedDate`の形式不正
- 最終確認日の未来日
- 資料公表日の形式不正・未来日
- 資料公表日が最終確認日より後になる矛盾

### 鮮度

基準日からの経過日数が次を超えた企業を検出する。

- 本番: 90日
- 詳細抽出済みβ: 180日
- 一次確認β: 365日
- Coverageβ: 365日

PRとmainの決定論的監査では`operations/quality-debt-budget-v1.json`の基準日を使う。毎週のスケジュール監査では実行日のUTC日付を使い、期限超過を自動検知する。

### 品質階層

- 本番の公表日・ページ証跡・数値・進捗・証跡不足
- 詳細抽出済みβの公表日・ページ証跡・数値・証跡不足
- 一次確認βに未レビュー分析が混入していないか
- Coverageβに戦略分析や中計情報が混入していないか
- CoverageβのJPX参照URLが証券コードと一致しているか

一次確認βの説明用テンプレートや「未抽出」という明示は分析情報として扱わない。Coverageβの企業探索用説明とJPX参照URLも正規のカバレッジ情報として扱う。

### 証跡・重複

- 空の証跡
- 同一企業内の重複証跡
- 異なる企業で同一の公式資料URL
- 異なる企業で同一の構造化サマリー

## CIの判定

実績件数が`maximumCounts`を1件でも上回るとCIを失敗させる。

実績件数が上限を下回った場合は改善としてレポートへ出力する。改善後は、同一PRまたは次の品質PRで上限を実績値まで引き下げる。

新しい監査項目がコードに追加され、予算ファイルに項目がない場合は上限0件として扱う。

## 自動処理の制限

品質負債ゲートは次を行わない。

- 未確認の資料公表日を推測して補完する
- ページ番号を推測する
- 企業を自動的に本番昇格する
- 品質スコアを企業評価や投資判断に利用する

## 成果物

GitHub Actionsは次を90日間保存する。

- `quality-debt-report-v1.json`
- `quality-debt-items-v1.csv`

JSONは集計・予算超過・改善項目を記録し、CSVは企業別の補修対象を記録する。
